# Token Saver 第一层规则（Phase 1）

状态：已验证稳定，可作为后续测试与调用的默认第一层规则。

## 规则目标
在尽量不影响模型稳定性的前提下，先进行最保守的上下文压缩测试。

## 当前规则
1. 只压缩旧轮次摘要（old context summary）。
2. 最近几轮对话原样保留。
3. 不改变 role 顺序。
4. 不修改 tool 相关结构。
5. 不改动真实用户当前输入内容。
6. 当模型错误频发时，可通过 wrapper 中的 Token Saver 手动开关立即关闭节流阀，回到稳定直通模式。

## 适用说明
- 这是 Token Saver 的第一层规则。
- 用于逐步收紧节流阀时的第一阶段测试。
- 后续更激进的压缩策略，必须在本规则稳定的前提下逐层增加。

## 观测要求
测试期间重点观察：
- 模型是否出现 error / aborted
- 同类输入是否开始随机失败
- dry-diff 中的：
  - messageCount
  - role 顺序
  - tool 信息
  - system message 是否改写
  - 最近三轮是否被动到

## 升级约束
只有在本规则稳定通过后，才允许进入下一层：
- 轻度压缩长 tool 输出
- 更强上下文折叠

## 回退约束
一旦出现频繁错误：
- 立即关闭 Token Saver 开关
- 保持 wrapper 稳定运行
- 通过 dry-diff + run-index 回查错误样本
- 分析是哪条压缩规则触发结构失真或模型错误
